# Start here

## You belong here

This lane welcomes curious people who are new to modular software, registries,
or working with AI agents. We introduce each idea before asking you to use it.
We treat confusion as useful feedback and explain the missing step.

*Made with love for creators of all kind.*

If you landed here with no context, begin with
[00: What even is all this?](00-orientation.md).

## What you will be able to do by lesson 18

By the end of this lane, you will be able to inspect a software portfolio,
recognize a SMARCH brick and its manifest, read registry and gate results, reuse
a module with its history intact, and coordinate agents with leases and conflict
records. You will also know how capsules, graphs, skills, canonical modules,
MCP connections, UltraVision plans, and agent swarms fit into the same system.

The goal is to help you make informed, verifiable changes without memorizing
SMARCH or keeping a framework expert beside you.

## Prerequisites

You need two tools:

- **Node.js**, to run the local tools and practice exercises.
- **Git**, to get the repository and keep track of changes.

Node.js and Git cover the starting lessons. Later lessons introduce optional
integrations when they become relevant.

## The no-stale-command promise

CI runs every command published in this lane against a practice portfolio. A
command that stops matching the tools or expected output fails the check. CI
catches stale instructions during the build, before you meet them halfway
through a lesson.

We publish no commands for lessons that have not landed. When we publish a
lesson, CI begins running its exercises as part of the verified journey.

Begin with [lesson 00: orientation](00-orientation.md).
