# Project metadata template

This README explains the project metadata template stored under `.sweetspot/` and the values a new project must replace. Project maintainers need it when they register a repository with SMARCH. Read it before copying the template or running the first project scan. Remember to replace both the project identity and module list before treating generated registry results as valid.

The two JSON files start with the same project shell so scanners can discover the repository before its module manifests exist. Keep their project fields aligned as modules are added.
