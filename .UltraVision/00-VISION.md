# SMARCH UltraVision — 00-VISION

Repo: `~/DEV/SMARCH` (public SMA Gen3 framework, github.com/B-EtterDigital/SMARCH).

Vision basis: the 2026-07-10 deep architecture audit (this session), repo docs, and user directives.

## One-line vision

SMARCH is the coordination and provenance substrate that makes fleets of AI
agents safe in real codebases — and it must feel so light, typed, and
demo-obvious that the agent-native crowd (the Lakebed/T3 audience) *wants* it.

## Vision pillars

### Foundation (existing, [source] — listed for confirmation)
- **V-01 — Brick contract & registry-with-rejection.** Every reusable module
  carries boundaries, security rules, tests, provenance, clone instructions;
  the registry only matters if bad bricks are rejected. [source: README.md:1-23,
  docs/FRAMEWORK.md, docs/PUBLIC_POSITIONING.md]
- **V-02 — Gen3 multi-agent coordination substrate.** Leases, append-only
  agent-context "why" logs, intent-level conflict reports, controller sweeps —
  portfolio-level coordination nobody else ships. [source:
  docs/MULTI_AGENT_OPERATIONS.md, tools/sma-lease.mjs, tools/sma-conflict.mjs]
- **V-03 — Provenance & license integrity chain.** in-toto Statement v1 + SLSA
  Provenance v1 attestations, tamper seals, license lattice, creator trails.
  [source: tools/sma-attest.mjs, tools/lib/license-lattice-selftest.mjs]

### Elevation (from the 2026-07-10 deep audit, [elevated])
- **V-04 — Graphify at full power.** Embedding index + semantic re-rank in
  `query`; real source→graph staleness in `graphify:check`; `global query`
  across the project union; GraphRAG community summaries consulted at query
  time; tree-sitter-grade extraction. [evidence: query.md:25 substring+IDF,
  sma-graphify.mjs:405-447 existence-only check, docs/CODEX_INTEGRATION.md:138
  embeddings TODO]
- **V-05 — Registry as MCP server + Server Card.** Brick search, trust
  inspection, doctor/why-blocked, and installRelease exposed as MCP tools so
  any agent in any harness can consume the registry; publish a Server Card.
  [evidence: audit §2.1; MCP H1-2026 registry standardization]
- **V-06 — Capsule-grade brick tier (constraint-first).** A strict scaffold
  (fixed entry points, no arbitrary deps, runtime-inspectable fixtures, one
  `sma brick run/inspect` CLI) where promotion gates pass by construction —
  the Lakebed lesson folded into SMA. [evidence: audit §1; docs.lakebed.dev]
- **V-08 — Full TypeScript migration + Rust core kernel (G1-upgraded, [user]).**
  Phased, never big-bang: M0 tsconfig + JSDoc/`checkJs` + `tsc --noEmit` CI
  gate; M1–M2 module-by-module conversion of all tools/lib to erasable-syntax
  `.ts` (runs natively via Node type-stripping — the zero-build clone-and-run
  property survives the migration); M2 schema-derived types; M3 oxidation:
  `smarch-core` Rust kernel (scan/hash/similarity) + single static binary
  distribution. [user ballot 2026-07-10: options "TS + Rust kernel" AND
  "Full TS rewrite" combined]
- **V-09 — SMOA workforce-backend abstraction.** Pluggable executor backends
  (codex/opencode/claude -p) behind one dispatch contract so model churn stops
  requiring skill surgery. [elevated]
- **V-10 — Public CI.** GitHub Actions running `npm run check`, gitleaks, and
  the private-identifier leak grep on every PR. [elevated]
- **V-11 — Agent-performance evals.** Regression evals for skills/workflows
  (seeded runs, scored outcomes) so agent behavior is measured, not assumed.
  [elevated]
- **V-12 — Claude-plugin packaging.** Ship the skills as an installable
  plugin/marketplace package (the Superpowers-proven distribution shape).
  [elevated]
- **V-14 — Ambient control plane.** Hooks/wrappers that auto-lease and
  auto-context so Gen3 ceremony becomes invisible plumbing — coordination
  without ritual. [elevated: philosophy-conflict resolution #2]
