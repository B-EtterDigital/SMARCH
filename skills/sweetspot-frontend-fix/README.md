# Sweetspot Frontend-Fix ([SFF](../../docs/GLOSSARY.md#sff))

This README explains the opt-in design process that creates a frontend direction, verifies the rendered result, and records a design lock. Design leads and frontend implementers need it after the user invokes Frontend-Fix or when a project carries a lock. Read it before changing a locked visual surface or starting the approval workflow. Remember that locked design decisions stay fixed until the user requests another Frontend-Fix pass.

See [SKILL.md](./SKILL.md) for activation rules, design research, anti-slop constraints, approval steps, and the delivery gate.
