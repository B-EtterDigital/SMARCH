# .UltraVision Folder Structure and Document Templates

`.UltraVision/` is the plan of record — read by humans *and* agents. Documents are precise, cited, visually excellent markdown: tables for enumerable facts, Mermaid for structure, zero filler. English for plan docs; the *product* gets the multilingual preparation.

## Required tree

```
.UltraVision/
├── INDEX.md                    GENERATED by `uvp render` (guidance goes in meta/index-notes.md)
├── 00-VISION.md                vision pillars V-xx with evidence, [source|elevated] tags
├── 01-CURRENT-STATE.md         cited audit
├── 02-GAP-ANALYSIS.md          gap register G-xx (the coverage contract)
├── 03-TARGET-ARCHITECTURE.md   target module map, seams, Mermaid, hot-path inventory
├── 04-DESIGN-LANGUAGE.md       design contract (per design-excellence.md)
├── 05-MEDIA-PLAN.md            asset plan + per-milestone paid batches, or reasoned N/A
├── 06-I18N-PLAN.md             locale architecture, extraction strategy, RTL, pseudo-locale
├── 07-PERFORMANCE-PLAN.md      named baseline device, budgets, adaptive ladder, SOTA tier
├── 08-QUALITY-RELEASE-PLAN.md  milestones M0..Mn with definition-of-done, release trains
├── modules.json                Gen3 binding (contract in sma-gen3-spec.md)
├── inventories/
│   └── <module>.<type>.json    one file per module×type (collision-free parallel writes)
├── tasks/
│   └── <module>.jsonl          SOURCE OF TRUTH — modified only via `uvp` or expansion
├── tasklists/
│   └── <module>/
│       ├── _MODULE.md          hand-written charter (below)
│       └── <domain>.md         GENERATED views
├── templates/                  optional project-local template overrides
└── meta/
    ├── pipeline-state.json     resumability checkpoints (below)
    ├── vision.json             G1 outcome: pillars {id,title,status:approved|vetoed,
    │                           origin:source|elevated|user}, non_goals — `[user]` pillars come
    │                           from G1's "what's missing?" round, auto-approved, drift-exempt
    ├── vision-sources.json     per-pillar evidence files + hashes (managed by `uvp drift`)
    ├── stats.json              GENERATED
    ├── validation-report.json  GENERATED
    ├── topo.json               GENERATED (waves, critical path)
    ├── wave.json               GENERATED (`uvp dispatch` parallel-agent manifest)
    ├── report.md               GENERATED (`uvp report` velocity/burndown/ETA)
    ├── audit-report.json       GENERATED (`uvp audit-claims` spot-check results)
    ├── featmap-proposals.md    GENERATED (`uvp featmap` SFM rows for delivered pillars)
    ├── sma.gen3.draft.json     GENERATED (`uvp gen3-draft` bootstrap artifact)
    ├── journal.jsonl           GENERATED (claim/complete/verify/obsolete audit trail)
    ├── index.sqlite            GENERATED (query cache — safe to delete anytime)
    ├── waivers.json            hand-written coverage waivers with rationale
    └── index-notes.md          hand-written execution guide, included into INDEX.md
```

All ten root docs always exist — a non-applicable section holds a short reasoned N/A, never an absent file (agents must not wonder whether missing means "N/A" or "unfinished").

## meta/pipeline-state.json (resumability contract)

```json
{"mode": "full", "budget": null, "updated": "2026-07-02T10:00:00Z",
 "phases": {
   "P0": {"status": "done", "ts": "..."},
   "P1": {"status": "done", "ts": "..."},
   "G1": {"status": "approved", "by": "user", "ts": "...", "notes": "V-07 elevation vetoed"},
   "P4": {"status": "done", "ts": "..."},
   "G3": {"status": "approved", "by": "user", "projected_tasks": 8412},
   "P7": {"status": "in-progress", "cursor": {"module": "ui", "batch": 3}},
   "P8.5": {"status": "in-progress", "clean_rounds": {"ui": 1, "core": 2}}
 }}
```

On every invocation: read this first; resume at the first phase not `done`/`approved`. Update it at every phase end and every P6/P7 batch. Gates (G1 vision, G2 module map, G3 volume) record who approved and any vetoes — G1's outcome also lands in `meta/vision.json`, which the validator enforces (vetoed pillars can never re-enter the plan silently).

## Document templates (headings are the contract)

**00-VISION.md** — north-star paragraph; `### V-01 · <one-line user promise> [source|elevated|user]` each with `evidence:` citations (`[user]` pillars cite the G1 conversation instead) and `perfection means:`; **Non-goals** (scope armor for 100k tasks); **Open questions** (conflicts + elevations awaiting G1).

**01-CURRENT-STATE.md** — Snapshot (date, commit, method, graphs used); Module inventory table (module | ownership | maturity absent→polished | evidence); one section per quality dimension (features, architecture, tests, telemetry/SRS, design system, a11y, i18n incl. hardcoded-string count, performance, docs, CI/gates, release readiness) — every claim cited; **Strengths worth preserving**.

**02-GAP-ANALYSIS.md** — `### G-01 · <title> [blocker|major|minor|polish]` with `blocks: V-xx | modules | current (cited) | target`; Coverage matrix table pillars × modules → gap IDs (every pillar row fully covered).

**03-TARGET-ARCHITECTURE.md** — target module table (module | responsibility | ownership | allowed_deps | hot-path borders); Mermaid seam/flow diagrams; registry/seam patterns over central glue; migration notes; exact proposed `sma.gen3.json` changes (the change itself = shared-hot-path task).

**04-DESIGN-LANGUAGE.md** — named creative direction rooted in the product (never "modern minimal SaaS"); type/color/spacing/motion systems with a11y notes; component inventory pointer; 3–7 signature clever-UI mechanisms; dark/light strategy. Written so a frontend agent implements any screen without inventing style. Reference the user's installed taste skills where fitting.

**05-MEDIA-PLAN.md** — classification verdict quoting `modules.json.product.media_class` + rationale. Visual products: asset inventory pointer, pipeline routing, style-bible anchors (generated first, deps of everything), naming/manifest conventions, **per-milestone paid batches with rough credit cost** for one-decision user approval. Utility products: the verdict paragraph, nothing else.

**06-I18N-PLAN.md** — locale architecture and key naming; extraction strategy with audited hardcoded-string count; ICU formats; RTL plan; Intl-based formats; pseudo-locale in dev builds + CI hardcoded-string gate; launch vs prepared languages; text-in-media strategy for visual products.

**07-PERFORMANCE-PLAN.md** — named minimum-spec device the product must feel great on; per-surface budgets (startup, TTI, input latency, frame time, memory, bundle/transfer); adaptive quality ladder down-tier; **SOTA enhancement tier** (headroom → delight: fidelity, physics, prefetch — gated on detection, never taxing the baseline); perf gates entering CI.

**08-QUALITY-RELEASE-PLAN.md** — milestone ladder (M0 foundation/bootstrap, M1 MVP, M2 beta, M3 1.0, M4+ vision-complete — adapt count per project) each with definition-of-done and required gates; testing strategy per layer; telemetry/SRS coverage requirements; release-train rules for shared hot paths; this project's definition of "release-ready".

## _MODULE.md charter (hand-written, one per module)

```markdown
# Module: core
- ownership: src/core/**            (mirrors modules.json / sma.gen3.json)
- lane-default: single-module · allowed_deps: shared-hot-path
- gates: pnpm typecheck:core, pnpm test:core, pnpm srs:audit
- graph: graphify-out/modules/core/graph.json   (or "missing — see UV-CORE-baseline-graph")
- hot-path borders: <which shared paths this module touches, if any>
- how to work here: uvp next --module core --tier <yours>; claim → gates → complete with evidence
```

## meta/index-notes.md (hand-written, included into INDEX.md by render)

Execution guide: which module lanes can run in parallel now (disjoint ownership), shared-hot-path ordering, the repo's current Gen3 agent ceiling (per sma-gen3-spec), paid-batch approval status; SMA-P1..P3 procedural attestations; **Known plan gaps** — whatever blocked exhaustiveness, stated plainly.
