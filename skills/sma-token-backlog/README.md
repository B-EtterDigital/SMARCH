# SMA Token Accounting and Backlog

This README explains the skill that records reuse savings, inherited imperfections, and downstream updates after brick reuse or release. Brick maintainers and release operators need it when they clone a brick, leave a gate incomplete, promote a brick, or bump a source version. Read it before writing a reuse receipt or propagating an update. Remember to record unresolved debt in the backlog with its owner and cost.

See [SKILL.md](./SKILL.md) for required commands, output paths, and propagation rules.
