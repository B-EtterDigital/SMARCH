# Course Pattern

The SMA course should be generated from registry data and real source facts.

Use this order:

1. What a brick is.
2. Why small files matter for humans and agents.
3. How isolation prevents blast radius.
4. How to inspect trust.
5. How to read provenance.
6. How to clone safely.
7. How gates decide canonical status.

Good course blocks:

- code or manifest excerpt
- plain-English translation
- flow diagram
- decision quiz
- glossary term
- "copy mistake" warning

Avoid:

- acronym dumps
- generic software lectures
- fake code snippets
- model-name worship
- long paragraph-only lessons
